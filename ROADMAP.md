# AI Quant Builder Roadmap

This is the public product roadmap for accelerator and startup-program review.

## Phase 1 - Pre-trade Strategy Report

Goal: help a user turn one investment idea into a readable strategy validation report.

Planned workflow:

1. User enters a plain-language investment idea.
2. The system extracts assets, market, entry logic, exit logic, holding period, and risk assumptions.
3. The system converts the idea into testable strategy rules.
4. The system runs historical backtest and drawdown analysis.
5. The system generates a report with strengths, weaknesses, and improvement suggestions.

## Phase 2 - Saved Strategy Library

Goal: let users compare ideas over time.

Planned workflow:

1. Save strategy reports.
2. Compare multiple strategy versions.
3. Track which changes improved or weakened the strategy.
4. Export or share reports.

## Phase 3 - AI Improvement Loop

Goal: help users improve their thinking, not simply generate trades.

Planned workflow:

1. Identify failure periods.
2. Suggest filters or sizing constraints.
3. Run revised strategy variants.
4. Compare original and improved reports.

## Phase 4 - Broker / Execution Research

Goal: explore whether execution can be added safely.

This phase is not part of the first public MVP. Any future execution feature should require:

- user confirmation
- broker permissions
- risk warnings
- jurisdiction-specific compliance review
- clear separation between research output and order execution

## Near-term Milestones

- Publish public landing page
- Collect first user interviews
- Build idea-to-report MVP
- Add saved strategy reports
- Add simple comparison view
- Prepare accelerator/demo application materials
